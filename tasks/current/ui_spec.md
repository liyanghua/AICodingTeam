# UI Spec: 市场分析洞察报告生成器

> 应用形态：`report_generator`，单页 SPA。布局：左侧节点进度条 / 中间工作区 / 右侧 Evidence 抽屉 / 顶部全局栏。

## 1. 全局布局

```
┌────────────────────────────────────────────────────────────────────┐
│ 顶部栏：应用名 │ 用例预设切换 │ run_id │ 导出报告 │ 帮助           │
├────────────┬───────────────────────────────────────┬─────────────┤
│            │                                       │             │
│ 左侧节点    │  中间工作区                              │ 右侧         │
│ 进度条     │  - 表单 / CSV 上传 / 计算中 / 结果        │ Evidence    │
│ (10 节点) │  - 当前节点状态                            │ 抽屉        │
│            │                                       │             │
├────────────┴───────────────────────────────────────┴─────────────┤
│ 底部状态栏：节点状态摘要 │ 降级横幅（如有）                          │
└────────────────────────────────────────────────────────────────────┘
```

## 2. 左侧节点进度条

来源：`workflow.dag.yaml` 节点顺序。

| 显示 | 含义 |
| --- | --- |
| 圆点 + 节点中文名 | 节点状态色 |
| 灰 | `pending_input` |
| 蓝 | `running` |
| 绿 | `ok` |
| 黄 | `degraded` |
| 红 | `failed` |

点击节点 → 中间工作区切换到该节点视图。

## 3. 中间工作区视图（按节点 type）

### 3.1 `define_scope`（表单）

8 字段表单，按 PRD §3.1 顺序。顶部三按钮预设：「新类目立项 / 老类目优化 / 价格带扩张」。
提交后跳到 `collect_top_products`。

### 3.2 `data_collect` / `compute*` / `multi_source_analysis` / `external_research`

通用三段式：

- 上：上传槽位（拖拽 CSV、显示已识别字段、显示别名归一化提示）。
- 中：计算状态（`running` 时显示进度，`degraded` 显示原因，`failed` 显示需要补的字段清单）。
- 下：结果区
  - 表格：从 `outputs/<schema_id>.json` 渲染，列对照 `output_schemas/*.json`。
  - 结论卡：每条带 `label`（来自 rule output_label）、`supporting_metrics`（小字）、`evidence_ids` 链。

### 3.3 `score_opportunities`

- 大数字卡：分数 + 档位（立项 / 测试 / 观察 / 不开发）。
- 雷达图（可选，CSS 实现）：6 项分项展示。
- 表格：分项明细 + 各维度 evidence_ids。

### 3.4 `generate_listing_plan`

- 卡片列表：每个机会一张卡，包含定价区间、卖点组合、视觉锚点、问大家应答策略、关键词布局。

## 4. 右侧 Evidence 抽屉

触发：任意结论 `evidence_ids` 上的链接。
布局：

```
┌─────────────────────────────────────┐
│ Evidence 抽屉          [关闭]       │
├──────────────┬──────────────────────┤
│ 时间线        │ 字段对照               │
│ - source_name │ key      | value     │
│ - fetched_at  │ ─────────┼─────────  │
│ - run_id      │ ...      | ...       │
└──────────────┴──────────────────────┘
```

关闭抽屉不重置工作区状态。

## 5. 降级横幅

任一节点为 `degraded` 时，底部出现黄色横幅：「跨平台维度未参与本次评分，机会评分仅基于其他 5 维度」。

## 6. 阻断对话框

某节点为 `failed` 时，对应卡片显示对话框：

```
需补：competitor_reviews_qa CSV
缺以下字段：review_text, painpoint_type, sku_name
[重新上传]  [取消]
```

字段列表来自服务端 schema 校验。

## 7. 导出报告

顶部右侧按钮「导出报告」永远存在。仅当 `score_opportunities` 为 `ok` 时可点击；否则灰显并提示原因。点击后下载 `final_report.md`。

## 8. 文案规范

- 标签：强爆款基因 ✅ / 趋势爆款基因 🚀 / 差异机会 💡 / 立项 / 测试 / 观察 / 不开发
- 计算中：`计算中…`
- 待输入：`等待上传 <data_requirements.id>.csv`
- 降级：`<节点中文名> 已降级：<原因>`
- 失败：`<节点中文名> 阻断：<原因>`

## 9. 响应式

- 1280px 及以上：三栏布局。
- 1024-1280px：右侧抽屉改为弹层。
- < 1024px：左侧进度条改为顶部水平条；中间工作区单列。

## 10. 不实现

- 暗色主题、国际化、账户系统、多 run 对比（v1 不在 customizations 中）。
- 复盘 / 推送 / 定时任务（属于 v2 范围）。
