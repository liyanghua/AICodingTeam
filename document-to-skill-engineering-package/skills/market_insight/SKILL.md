# Market Insight Skill

## Purpose

将电商市场洞察策略转化为可执行分析流程，用于新品开发、爆款机会挖掘、类目市场分析、竞品分析、价格带布局、跨平台趋势机会分析和产品开发链接规划。

## When to use

- 需要判断一个类目是否值得做。
- 需要寻找爆款机会。
- 需要拆解关键词需求结构。
- 需要分析竞品、价格带、评价痛点。
- 需要输出新品开发与链接规划表。

## Required inputs

- `category`：类目，建议精确到三级类目或最小叶子类目。
- `product_line`：产品线。
- `analysis_period`：7d / 30d / monthly / seasonal。
- `business_goal`：新品开发 / 爆款机会挖掘 / 类目市场分析 / 竞品突破 / 产品升级 / 价格带布局。
- `shop_stage`：新店 / 成长期店铺 / 爆款店铺 / 多链接店铺。
- `target_price_band`：低价 / 中价 / 高价 / 生意参谋价格带。

## Workflow

1. 确定分析边界。
2. 获取行业大盘与热销商品数据。
3. 提炼爆款基因。
4. 获取并拆解关键词需求。
5. 获取评价与问大家痛点。
6. 分析价格带与利润空间。
7. 分析竞品与竞店格局。
8. 获取跨平台趋势机会。
9. 进行机会判断与产品创新设计。
10. 输出产品开发与链接规划。

## Data policy

- 优先使用内部数仓 API。
- 其次使用 BI API 或平台导出工具。
- 再其次使用浏览器自动化工具。
- 外部公开趋势可使用 Exa / Web Intelligence。
- 没有数据时不得编造经营结论。

## Evidence policy

- 每个结论必须有 Evidence Pack。
- 每个分数必须有公式或规则。
- 每个推荐必须能追溯到数据、规则或人工复核。

## Output policy

必须输出：

1. 类目大盘分析表。
2. 行业前300商品分析表。
3. 关键词需求拆解表。
4. 关键词词根TOP20表。
5. 评价/问大家痛点表。
6. 价格带机会表。
7. 竞品竞争格局表。
8. 竞品主图/详情页/客服对比表。
9. 跨平台趋势素材表。
10. 产品开发与链接规划表。

## Failure handling

- 内部 API 失败：尝试 BI API。
- BI API 失败：尝试浏览器导出。
- 浏览器失败：生成人工上传任务。
- 数据不完整：标记低置信度，不输出强结论。
