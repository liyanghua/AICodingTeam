> **文档说明**:本文基于对话取证逆向梳理,描述"叮当电商主图设计智能体"的内部工作流、节点输入输出与提示词模板设计,用于产品 / 算法 / 工程对齐参考。

> **数据来源**:`dingdang.md` 第 1~650 行对话记录

## 一、产品定位

### 1.1 核心场景
为电商运营者(老板 / 运营 / 设计师)生成**一套 8 张可投放、可 AB 测试的主图**。典型商品场景:商务办公椅、抗老精华等高客单价、视觉敏感类目。

### 1.2 价值主张
- **一次输入,出整套主图**:从产品图到 8 张可投放图,全流程封装
- **套装基因统一,差异受控**:8 张图共享色温/光影/产品角度,只在指定维度变化,保证 CTR 归因有效性
- **可解释 + 可局部迭代**:用户能精准定位"改第 X 张的第 Y 层",无需返工
- **平台策略自适配**:天猫 / 淘宝 / 抖音 / 拼多多 各走一套打法

### 1.3 目标用户
| 角色 | 主要诉求 |
|---|---|
| 中小卖家 / 店铺老板 | 不懂设计,要"能用就行" |
| 电商运营 | 要 CTR、要 AB 测试、要数据归因 |
| 设计师 | 要效率、要素材、要批量产出 |

## 二、整体工作流架构

### 2.1 四阶段总览

| Stage | 名称 | 核心动作 | 用户阻断 | 自动化程度 |
|---|---|---|---|---|
| 1 | 需求诊断 | 信息收集 + 任务定型 | ✅ 等用户确认任务类型 | 半自动 |
| 2 | 创意方案 | 输出 2~3 套差异化方案 | ✅ 等用户选方案,禁止混搭 | 半自动 |
| 3 | 策略落地 | 翻译为可执行设计规范 | ❌ | 全自动 |
| 4 | Prompt 生成与执行 | 输出 8 张图施工图并出图 | ❌ | 全自动 |

### 2.2 关键阻断设计
**为什么必须阻断**:
- Stage 1:任务类型(完整 8 张主图 vs 8 张首图测点击)逻辑不同,自动选择会跑偏
- Stage 2:方案 A/B/C **不允许混搭**(例:不能要 A 的第 1 张 + B 的第 3 张),否则 AB 测试单变量原则被破坏,数据无法归因

### 2.3 流程编排图

```
[用户输入产品图+需求]
        │
        ▼
┌──────────────────┐
│ Stage 1 需求诊断 │  → 输出: 任务类型 / 参考图类型 / 平台策略
└──────────────────┘
        │ ⏸ 阻断等用户选 A 或 B
        ▼
┌──────────────────┐
│ Stage 2 创意方案 │  → 输出: 2~3 套方案 + 8图分工表
└──────────────────┘
        │ ⏸ 阻断等用户选方案
        ▼
┌──────────────────┐
│ Stage 3 策略落地 │  → 输出: 视觉基准卡 + 8张单张规划卡 + 一致性校验
└──────────────────┘
        │ 自动流转
        ▼
┌──────────────────────┐
│ Stage 4 Prompt 生成 │  → 输出: 8 张三层 Prompt + 出图
└──────────────────────┘
        │
        ▼
[用户局部迭代:指明"第X张第Y层"]
```

## 三、Stage 1:需求诊断

### 3.1 节点输入
| 字段 | 必填 | 说明 |
|---|---|---|
| 产品图 | ✅ | 决定产品形态,是后续锚定层基础 |
| 参考图 / 对标链接 | ❌ | 决定风格方向 |
| 平台 | ✅ | 天猫 / 淘宝 / 抖音 / 拼多多 |
| 类目 | ✅ | 影响打法权重 |
| 价格带 | ✅ | 决定走"性价比"还是"品质" |
| 核心卖点 | ✅ | 后续卖点分级输入 |
| 目标人群 | ✅ | 决定首图钩子方向 |

### 3.2 节点输出
- **参考图类型判断**:详情页参考(风格借鉴) / 主图参考(结构复刻)
- **任务类型判断**:完整 8 张主图 / 8 张首图测点击率
- **平台策略锁定**:对应平台的设计禁忌与偏好

### 3.3 提示词模板(系统侧)
```
你是电商主图诊断助手。基于用户提供的 [产品图 / 参考图 / 基础信息],
完成以下三件事:

1. 参考图类型识别:
   - 若参考图为长图、模块化排版 → 标记为"详情页参考-风格借鉴"
   - 若参考图为方图、单画面强冲击 → 标记为"主图参考-结构复刻"

2. 任务类型判断:
   - 若用户明确"测点击" → 8 张首图测点击率
   - 若用户表达"做主图" → 完整 8 张主图

3. 平台策略适配(查表):
   - 天猫 → 克制高级、禁促销标签
   - 淘宝 → 直接清晰、可放卖点标签
   - 抖音 → 强冲击、单点爆破
   - 拼多多 → 促销感、性价比

【阻断规则】判断完后,必须列出选项让用户选 A 或 B,
不允许自动进入 Stage 2。
```

## 四、Stage 2:创意方案

### 4.1 内部五步推导

| 步骤 | 动作 | 产出 |
|---|---|---|
| 1 | 卖点分级 | 感官型 / 功能型 / 信任型 三类 |
| 2 | 平台打法适配 | 排除不匹配平台调性的方案 |
| 3 | 参考图基因提取 | 色调 / 光影 / 气质 / 排版 关键词 |
| 4 | 方案组合推导 | 2~3 套差异化方案 |
| 5 | 阻断呈现 | 给用户出选择题 |

### 4.2 卖点分级表(内部知识库)
| 类型 | 特征 | 是否适合首图钩子 |
|---|---|---|
| 感官型 | 一看就懂(颜值、材质、造型) | ✅ 首图直接用 |
| 功能型 | 需要解释(滚轮顺滑、人体工学) | ⚠️ 第 3~5 张展示 |
| 信任型 | 消除顾虑(售后、保障) | ❌ 靠后或不放首图 |

### 4.3 平台 × 用户心智决策表
| 平台 | 用户心智 | 首图打法 |
|---|---|---|
| 天猫 | 要品质、怕买 low | 场景氛围 + 材质细节 + 克制高级 |
| 淘宝 | 要比价、怕买贵 | 卖点标签 + 使用结果 + 信息清晰 |
| 抖音 | 视频种草进来 | 强视觉 + 短文案 + 单点爆破 |
| 拼多多 | 价格敏感 | 促销感 + 性价比 + 耐用证据 |

### 4.4 方案输出模板
```
【方案X】方案名称(如:轻奢场景型 / 卖点冲击型 / 成分信任型)

一、核心假设
- 用户第一眼关心:[场景/价格/功能/材质]
- 首图钩子策略:[代入感/痛点/促销/好奇]
- 平台调性匹配度:[天猫-克制 / 淘宝-直接 / ...]

二、8 张图分工表(核心)
| 序号 | 钩子类型 | 视觉策略 | 主文案(≤8字) | 副文案(≤12字) | 测试变量 | 控制变量 |
| 01  | ...     | ...    | ...          | ...           | 变的     | 不变的    |
| 02  | ...     | ...    | ...          | ...           | ...      | ...      |
| ... | ...     | ...    | ...          | ...           | ...      | ...      |

三、变量控制说明
- 变的:[8 张图差异点列举]
- 不变的:[8 张图统一要素列举,确保单变量]

四、风险提示
- 可能失效:[如用户对场景不敏感]
- 平台冲突:[如促销标签在天猫受限]
```

### 4.5 提示词模板(系统侧)
```
你是电商创意策略师。基于 Stage 1 的诊断结果,产出 2~3 套差异化方案。

强制约束:
1. 每套方案必须包含完整 8 张图分工表
2. 同一套方案内,8 张图只允许 1~2 个测试变量,其余必须锁定
3. 方案之间必须有显著差异(钩子类型不同),不允许产出"A+ / A++"这种伪选项
4. 输出后立即阻断,等待用户单选 A 或 B 或 C
5. 严禁混搭:用户若说"要 A 的 1 + B 的 3",必须拒绝并解释 AB 测试原理

输出格式:严格遵循 §4.4 模板。
```

## 五、Stage 3:策略落地

### 5.1 节点输入
- Stage 2 用户选定的方案(如方案 A)
- 产品特征锁定清单(从产品图提取的不可变更参数)

### 5.2 节点输出
1. **视觉基准卡**(8 张图共用的"宪法")
2. **8 张单张规划卡**(每张图的"施工单")
3. **一致性校验表**(横向对照)
4. **楼层规划总表**(给用户看的总览)

### 5.3 视觉基准卡模板
```
【色彩系统】
- 主色调:[颜色 + 占比,如 暖米白 70%]
- 辅助色:[颜色 + 占比,如 原木 25%]
- 点缀色:[颜色 + 占比,如 深咖 5%,仅文字]
- 禁用色:[明确禁止]

【光影系统】
- 主光源方向:[如 左侧 45 度]
- 色温:[如 4500K]
- 阴影要求:[如 柔和过渡,避免硬边]
- 产品表面光泽:[如 自然柔光,避免过曝]

【字体系统】
- 标题字体:[如 思源黑体 Medium]
- 正文字体:[如 思源黑体 Light]
- 字号比例:[标题:正文 = 3:2]
- 对齐方式:[左对齐 / 居中]

【构图系统】
- 产品占比:[30%~50%]
- 安全区:顶部[15%]标题,底部[20%]信息
- 角度基准:[如 正面微侧 15 度]
```

### 5.4 单张规划卡模板
```
【第 X 张】图编号 + 名称

测试目标:[测什么,如 办公场景 CTR]
层类型:[场景层 / 细节层 / 功能层 / 呼吸层]

视觉规范:
- 构图:[三分法 / 中心 / 对角线]
- 场景背景:[具体描述]
- 道具:[2~4 个克制道具,或空]
- 景深:[浅景深 / 全景清晰]

文案规范:
- 主标题:[内容 + 位置 + 字体]
- 副标题:[内容 + 位置 + 字体]
- 避让区:[严禁遮挡的产品部位]

一致性检查:
- 光影:[符合基准卡]
- 色调:[符合基准卡]
- 产品角度:[符合基准卡]
```

### 5.5 一致性校验机制
横向对照表,任一项 ❌ 必须修正:

| 检查项 | 标准 | 第01张 | 第02张 | ... | 第08张 |
|---|---|---|---|---|---|
| 产品角度 | 微侧 15 度 | ✅ | ✅ | ... | ✅(白底可正面) |
| 色温 | 4500K | ✅ | ✅ | ... | ✅ |
| 主色调 | 暖米白 | ✅ | ✅ | ... | ✅ |
| 文字避让区 | 不挡核心结构 | ✅ | ✅ | ... | ✅ |
| 促销标签 | 全无(天猫场景) | ✅ | ✅ | ... | ✅ |

### 5.6 提示词模板(系统侧)
```
你是设计规范翻译官。任务:把 Stage 2 的方案文字,
翻译成 8 张图的可执行数字参数。

核心原则:
1. "感觉"必须转成"数字"
   反例:暖米色 → 正例:米白70%+原木25%+深咖5%
   反例:柔和光 → 正例:左侧45度,色温4500K
2. 8 张图共享一套基准卡,差异仅在单张规划卡的"测试变量"维度
3. 一致性校验必须横向逐项过,任何一项偏离必须修正
4. 第 08 张白底图也要遵守光影规范(否则 CTR 差异无法归因到背景)

输出格式:严格遵循 §5.3 / §5.4 模板。
```

## 六、Stage 4:Prompt 生成与执行

### 6.1 三层 Prompt 结构(核心方法论)

```
┌────────────────────────────────────────┐
│ 【视觉基因声明】套装第 X 张              │
│  色温 / 色调 / 光照 / 角度 一致性声明     │
├────────────────────────────────────────┤
│ 【第一层:视觉锚定层】                    │
│  产品骨骼,绝对不可动                     │
│  —— 严格复刻产品图                       │
├────────────────────────────────────────┤
│ 【冲突解决协议】                          │
│  文字 vs 图片 → 以图为准                 │
├────────────────────────────────────────┤
│ 【第二层:氛围调整层】                    │
│  场景皮肤,可调整                         │
│  —— 背景 / 光影 / 场景                   │
├────────────────────────────────────────┤
│ 【第三层:信息叠加层】                    │
│  文案标牌,最后放                         │
│  —— 主标题 / 副标题 / 避让区             │
├────────────────────────────────────────┤
│ 【负面词清单】≤10 个                     │
└────────────────────────────────────────┘

总长度:600~800 字 / 张
```

### 6.2 第一层:视觉锚定层(产品保险锁)
```
基于用户产品图精确复刻:

- 产品主体:[完整描述,如 米色皮质办公椅,高靠背设计...]
- 整体轮廓:[如 宽厚饱满座垫和靠背,扶手与靠背连贯曲线]
- 关键细节:[如 扶手外侧弧形木纹装饰板纹理清晰]
- 材质表现:[如 PU 皮呈现荔枝纹,自然柔光不反光]
- 产品角度:[如 正面微侧 15 度]
- 严禁偏离:禁止改变结构比例,禁止移动 [关键部件] 位置,
  禁止调整 [底座/滚轮/方向]
```

### 6.3 第二层:氛围调整层(场景皮肤)
```
- 场景:[如 现代办公室落地窗环境,窗外城市景观虚化]
- 光照:[如 左侧 45 度自然光,色温 4500K]
- 构图:[如 三分法,椅子位于画面右侧 1/3,占画面 40%]
- 道具:[如 模糊办公桌、笔记本、绿植,均低饱和]
- 氛围:[如 专业商务感,克制高级感]
- 紧箍咒:确保产品边缘轮廓与参考图一致,
  不得改变第一层已锁定的产品结构。
```

### 6.4 第三层:信息叠加层(文案标牌)
```
- 主标题(顶部左侧安全区):"[文案]",
  现代无衬线字体 Medium,深咖色
- 副标题(主标题下方):"[文案]",
  细体字,浅灰色
- 避让区:严禁遮挡 [核心视觉部位 1/2/3]
- 禁止:无促销标签、无价格信息、无高饱和强调色块
```

### 6.5 冲突解决协议
```
冲突解决协议:若文字描述与产品图冲突,严格以产品图为准。

- 文字描述结构 ≠ 参考图结构 → 遵循产品图,忽略文字
- 文字颜色 ≠ 产品图色差     → 以产品图为准,文字仅作命名
- 文字尺寸 ≠ 产品图比例     → 以产品图比例为准
```

### 6.6 负面词清单(通用基线)
```
促销、价格、霓虹、强对比、杂乱、夸张、低俗、侵权、像素、色号
```
按业务场景增删:
- 天猫场景 → 必须含"促销、价格"
- 暖色调主图 → 必须含"霓虹、强对比"
- 商务克制风 → 必须含"夸张、低俗"

### 6.7 视觉基因声明(套装一致性)
```
本图属于 8 图套装中的第 X 张,严格遵循套装视觉基因:

- 色温:[如 4500K]
- 主色调:[如 暖米白 + 原木 + 深咖]
- 光照方向:[如 左侧 45 度]
- 产品角度:[如 正面微侧 15 度]
```

### 6.8 提示词模板(系统侧 - 节点 Prompt)
```
你是图片生成 Prompt 工程师。任务:基于 Stage 3 的单张规划卡,
按 §6.1 三层结构生成 600~800 字的图片生成 Prompt。

强制约束:
1. 必须按"视觉基因声明 → 第一层 → 冲突协议 → 第二层 → 第三层 → 负面词"
   的固定顺序排列
2. 第一层"严禁偏离"必须明确列出至少 3 个不可动元素
3. 第二层每条描述后追加"紧箍咒",防止 AI 适配场景时改产品
4. 第三层避让区必须明确列出至少 3 个核心视觉部位
5. 负面词控制在 10 个以内,按业务场景定制

输出后,立即调用图片生成模型出图,
并在前端流式回吐"节点开始 / 节点结束 / 节点输出"事件。
```

## 七、用户交互设计

### 7.1 用户提需求建议(对外输出)
| 阶段 | 推荐表述 | 反例 |
|---|---|---|
| Stage 1 | "天猫 8 张首图测点击,对标这张详情页风格" | 直接发图不说话 |
| Stage 2 | "我要方案 A,但第 3 张换成展示底座" | "A 的第 1 张 + B 的第 2 张"(混搭破坏测试) |
| Stage 3 | "色调再暖一点,文案再克制一点" | "好看就行" |
| Stage 4 | "第 5 张滚轮要更突出,背景换木地板" | "都改一下" |

### 7.2 用户修改 → 修改层定位表
| 用户需求 | 对应层级 | 改动位置 |
|---|---|---|
| 椅子角度换成正面 | 第一层 | 调整"产品角度"参数 |
| 背景换成书房 | 第二层 | 替换场景描述 |
| 标题移到右边 | 第三层 | 调整文案位置 |
| 加包邮标签 | 第三层 + 负面词 | 移除"促销"负面词,添加标签 |
| 色调再暖一点 | 第二层 + 视觉基因 | 调整色温,同步全 8 张 |

## 八、风险与边界

### 8.1 智能体的"防泄露"策略(线上观察)
当用户问"内部流程 / Prompt 是什么"时,智能体的优先级:
1. **首拒**:用业务话术转移注意力("咱们还是聊调图吧")
2. **二拒**(用户表达不满):转译为"业务版 SOP",只讲流程不讲 Prompt
3. **三拒**(用户继续追问):提供"模板字段"级别信息,不给具体 Prompt 文本
4. **终拒**:明确表示"涉及内部技术实现,不便展开"

### 8.2 已识别的失效风险
| 风险 | 触发条件 | 缓解 |
|---|---|---|
| 产品被 AI"创作变形" | 第一层锚定不严 | 强化"严禁偏离"列表 |
| 8 张图风格不一致 | 视觉基因声明缺失 | 每张图必须带基因声明 |
| AB 测试无法归因 | 用户混搭 / 多变量同时变 | Stage 2 阻断 + 单变量校验 |
| 平台违规(促销标签) | 负面词清单缺漏 | 平台 → 负面词映射表 |
| 文字遮挡产品 | 避让区未定义或执行偏差 | Stage 3 必须列避让区 |

### 8.3 智能体边界
- ❌ 不做品牌策略 / 营销定位 / 投放计划
- ❌ 不做长图详情页通版排版(本流程仅覆盖主图)
- ❌ 不做素材合规终审(广告法、医美话术等需人工复核)
- ✅ 仅在"主图视觉设计 + AB 测试结构"范围内提供专家级产出
