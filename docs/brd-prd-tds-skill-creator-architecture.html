从业务需求到应用生成主链路

BRD 先转成 PRD；PRD 再与业务能力、应用规范和评估标准一起编译成 TDS Bundle（Code Agent 输入包），由 Code Agent 生成可验证应用。

BRD PRD 能力上下文 应用规范 评估规范 TDS Bundle（Code Agent 输入包） Code Agent

极简架构图

主链路只有一个交付包：所有生成输入先进入 TDS Builder，再统一输出 TDS Bundle（Code Agent 输入包）。

模块说明

BRD

业务需求文档。定义业务目标、目标用户、成功指标和业务边界，是产品需求的来源。

PRD

产品需求文档。把 BRD 转成用户流程、功能范围、页面状态和验收标准。

Capability Context

业务能力上下文。补充数据、知识、工具、样例、反例和业务风险。

AppSpec

应用规范。定义模板、UI、技术栈、组件槽位、运行边界和功能设计。

EvalSpec

评估规范。定义 hard gates、评分维度、验证命令、安全规则和质量门禁。

TDS Builder

编译器。把 PRD、Capability Context、AppSpec、EvalSpec 合成 TDS Bundle（Code Agent 输入包）。

TDS Bundle（Code Agent 输入包）

唯一交付包。包含产品范围、能力投影、应用规范、任务拆解、验收映射和验证命令。

Code Agent

应用输出口。只读取 TDS Bundle（Code Agent 输入包），生成应用并返回验证结果和风险。

设计原则

先产品化

BRD 先变成 PRD，业务语言先转成产品语言。

四类输入

TDS Builder 只接收 PRD、Capability Context、AppSpec、EvalSpec。

单一输入包

Code Agent 只读取 TDS Bundle（Code Agent 输入包）。

AppSpec 前置

模板、UI、技术栈和功能边界先进入 TDS Bundle（Code Agent 输入包）。

EvalSpec 前置

质量门禁先进入 TDS Bundle（Code Agent 输入包），生成结果才可验证。

最终口径：BRD -> PRD；PRD + Capability Context + AppSpec + EvalSpec -> TDS Builder -> TDS Bundle（Code Agent 输入包） -> Code Agent -> Generated App。
