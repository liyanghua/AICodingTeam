能力运行时 · 生态飞轮

四角色 × 入口 × 贡献 × 分润

为什么这套架构天然能长成生态。四类电商既有玩家——培训老师、TP 代运营、数据服务商+平台、商家——各自手里的资产，正好对应 Skill 的四个面（骨架 / SOP+阈值 / 数据 / evidence）。每类角色都有自然的入口、明确的贡献和复利型分润，飞轮转一圈，下一圈更省力。

飞轮总览

中心是 Skill 资产池（骨架 + SOP+阈值 + 数据 + evidence 四面）。四角色顺时针围绕，编号 1–6 是飞轮六圈的顺序。

读图：实线（1–3、5）是显性资产贡献；虚线（4、6）是隐性放大效应——切片细化让骨架变出 N 个变体，分润透明吸引更多老师入驻。

四角色 × 入口 / 贡献 / 分润

培训老师

方法论骨架作者

入口
document-to-skill 编译器（外部 SDK / 上传后台）把方法论文档编译成 Strategy IR 与 Workflow DAG
贡献
骨架 Skill 的判断逻辑、节点编排与知识引用参数留空，由下游角色补
分润机制
按 Skill 调用次数分成（含派生变体）骨架被切成 N 个变体后，每个变体仍记原作者份额

TP 代运营商

SOP 与圈层阈值校准者

入口
SOP 补丁库 / 圈层阈值上传绑定到某个骨架 Skill 的特定圈层切片
贡献
Workflow 补丁、节点阈值经验值、节奏 SOP让 Skill 在该圈层下表现更准
分润机制
所属圈层变体的调用分成商家越多落在该圈层、TP 收益越高

数据服务商 + 平台

数据供给与切片维度

入口
Data Connector 注册中心（按 Tool Contract）声明字段、口径、刷新频率与权限
贡献
行业大盘、竞品、流量、交易数据 + 圈层切片字段让"圈层化"成为可能
分润机制
按 Skill 调用时的数据访问次数计费数据从"卖一次性看板"变成"按调用收水电费"

商家

运行期 evidence 与反馈末端

入口
用生成的 App 跑业务无需感知 Skill / 工件存在
贡献
trace、evidence、决策反馈（按 Skill 聚合、匿名化）反向回灌上游作者校准变体
收益
免费/低价用上越来越准的 Skill无需贡献现金，贡献的是匿名运行数据

飞轮六圈：每一圈让下一圈更省力

  1. 老师入驻 → 骨架 Skill 上架

    把方法论编译成裸 Skill，参数留空。冷启动靠老师自带的学员/受众，强行启动第一批商家。

  2. 商家试用 → 运行期 evidence 流出

    App 跑业务产出 trace / evidence / 反馈，按 Skill 维度聚合并匿名化，成为校准上游的"群体经验"。

  3. TP 看到缺口 → SOP 补丁与阈值校准

    TP 看得懂 evidence，针对评分偏低或漏判的圈层贡献补丁，Skill 准度提升。

  4. 切片细化 → 一个骨架长出 N 个变体

    数据维度丰富后，圈层粒度变细（品类 × 客单 × 流量 × 阶段），骨架被复用 N 次，老师复利 N 倍。

  5. 数据服务商接入 → 按调用收费

    调用量起来后，数据服务商按 Tool Contract 把数据接进来，让圈层切片字段持续扩充。

  6. 更多老师入驻 → 回到第 1 圈，起点更高

    Skill 分润透明、复利可见，吸引更多骨架贡献者；新老师立刻享受现成的数据 / SOP / 切片基础设施。

三道叠加的护城河

密度

Skill 密度护城河

骨架 + SOP 补丁越多，可执行业务场景越密。竞品追赶要补齐整张网，不是补一个功能。

最厚

Eval 校准护城河

每个 Skill 跑越多次，圈层参数校准越准。只能靠时间积累，资金和人力都补不齐。

粒度

圈层切片护城河

切片越细，变体越多。新入场者只能做"通用版"，打不过"精细版"。

冷启动卡点：飞轮转起来不能破的三件事

冷启靠老师，不靠商家

商家没动力试用没人背书的裸 Skill。老师自带学员能强行启动第一批，跳过了零调用陷阱。

evidence 必须按 Skill 聚合 + 匿名化

反馈不回到上游作者，TP 没缺口可补，飞轮第 3 圈卡死。匿名化是数据合规与共享的前提。

trace 必须资产标签可归属

每次调用要算清楚分润给老师 / TP / 数据服务商各自多少，否则上游不愿贡献。这是分润机制的硬条件。

架构对位：开放面已经在那儿

这套生态不需要重写架构。现有 capability-runtime + Skill 工件链就是开放面，把内部门变成外部门即可。

document-to-skill 编译器

→ 外部 SDK / 老师上传入口，骨架方法论的注册口

Tool Contract / Data Requirement

→ 数据服务商注册中心，统一字段与计费口径

Skill Spec 加圈层切片元数据

→ TP 补丁通道，绑定骨架到特定圈层

运行期 trace / evidence

→ 按 Skill 聚合的反向回灌通道，校准上游变体

第 0 天每个工件就设计成 schema-first、契约清晰、可回放——生态接口已经预留好，只需要打开。
