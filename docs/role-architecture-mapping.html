技术架构 × 产品设计 → 生态角色对应一页纸

把"App 壳 (A) + 能力运行时 (B) + 证据/校准回路"的每一层落到四类生态角色 + 我们自己，明确谁是主产出、谁是主消费、谁有权属。 横切的产品设计入口在每张角色卡里给出。规则只有一条：权属归平台的工件不可外带，能力签名跨阶段不变。

●主产出 ▲主消费 ◎产消双向 ◆权属/治理 —不参与

1. 全景：技术架构层 × 角色着色

上半生成期、下半运行期；A 壳左、B 能力运行时右；底部为反馈/校准回流。色块颜色 = 该层主要贡献角色。

行 = 技术架构 / 产品设计中的工件；列 = 生态角色。每格回答："这块谁产、谁用、谁有治理权"。

  • 方法论可带走，Evidence/校准参数不可带走

  • 多店切换 + 批量执行

4. 四条不变量：跨阶段不动

① 能力签名不变

裸服务端点 → Skill 包，调用契约一致；A 壳不感知后端是哪种封装。

② A/B 物理可合体、逻辑必分离

MVP 1.0 同进程；后续迁出 = 搬迁不是重写。A 共享"定义"、B 共享"执行+状态+证据"。

③ Trace 不可跳过

每一次场景端点调用都必须落 trace；无 trace 的结论不允许进入 Evidence Pack。

④ 权属归平台的工件不可外带

方法论可带走，证据 / 校准参数 / 切片样本不可带走。合作社区协议直接写入这一条。
