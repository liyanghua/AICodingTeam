# App Generation Deterministic Fallback Spec

## 状态

本规范定义阶段 0a：deterministic 模式下的兜底生成器，当 `--executor deterministic` 时产出一个最小可运行的 SPA 模板，而不依赖外部 codex 二进制。

## 目标

让 `app generate` 在**零外部依赖**（无 codex 二进制、无 LLM 调用）时仍能产出一个真实可运行的本地应用，保证"PRD → 能跑的实例"价值闭环在任何环境下可复现。

Codex 路径作为高质量生成叠加在上面，deterministic 路径是兜底保证。

## 技术形态

- 前端：原生 SPA（`index.html` + `styles.css` + `app.js`）
- 服务端：Node stdlib HTTP server（`server.js`）
- 持久化：`localStorage`
- 数据库：无
- 模板风格：简洁、无框架、无构建工具、零 npm 依赖

## 输入

- `prd_text`：PRD 文本内容
- `app_slug`：应用标识符（符合 `[a-z0-9-]` 模式）
- `app_contract.json`：已生成的应用契约（包含 `generated_app_dir`、`preview` 等）

## 输出

在 `generated_apps/{app_slug}/` 下生成以下文件：

1. `server.js`：Node stdlib HTTP 服务器，监听 `preview.url` 中定义的端口
2. `public/index.html`：SPA 入口，包含 `#app` 挂载点和基础 meta 标签
3. `public/styles.css`：最小样式（布局、表单、按钮、列表基础样式）
4. `public/app.js`：核心逻辑，读写 `localStorage['{app_slug}-state']`，渲染空状态/加载/错误/成功四种状态
5. `README.md`：本地运行说明

## 模板内容约束

### `server.js`

- 使用 Node stdlib `http`、`fs`、`path` 模块
- 从 `app_contract.json` 的 `preview.url` 中提取端口（默认 8788）
- 提供静态文件服务，根路径映射到 `public/index.html`
- 404 响应明确文本
- 支持 `.html`、`.css`、`.js` MIME 类型
- 启动时打印 `Server running at {preview.url}`

### `public/index.html`

- `<!doctype html>` + `<html lang="zh-CN">`
- `<meta charset="utf-8">` + `<meta name="viewport" content="width=device-width, initial-scale=1">`
- `<title>` 从 PRD 第一个非空行提取（fallback 到 app_slug）
- `<link rel="stylesheet" href="styles.css">`
- `<div id="app"></div>` 作为挂载点
- `<script src="app.js"></script>`

### `public/styles.css`

- CSS reset（`box-sizing: border-box`、`margin: 0`、`padding: 0`）
- 基础布局（`.container`、`.card`）
- 表单控件（`input`、`button`、`select`）
- 状态指示（`.empty-state`、`.loading`、`.error`）
- 列表样式（`.item-list`、`.item`）
- 符合 DESIGN.md 的 color tokens（可选，暂用中性灰蓝调色板）

### `public/app.js`

- 从 `localStorage.getItem('{app_slug}-state')` 读取状态（默认 `[]` 或 `{}`）
- 提供四种状态渲染函数：
  - `renderEmpty()`：空状态提示
  - `renderLoading()`：加载中
  - `renderError(message)`：错误展示
  - `renderSuccess(data)`：主界面
- 持久化：任何状态变更后 `localStorage.setItem('{app_slug}-state', JSON.stringify(state))`
- 暴露全局 `window.app = { state, render }` 便于浏览器控制台调试

### `README.md`

```markdown
# {app_slug}

Generated by Agent Team Runtime (deterministic fallback).

## Run Locally

\`\`\`bash
node server.js
\`\`\`

Open http://127.0.0.1:8788 in your browser.

## PRD Summary

{从 PRD 提取的第一段或前 200 字符}
```

## 生成逻辑位置

新增函数：`growth_dev/team/app_generation.py::generate_deterministic_app_files()`

签名：
```python
def generate_deterministic_app_files(
    *,
    run_dir: Path,
    app_slug: str,
    prd_text: str,
    contract: dict[str, Any],
    repo_root: Path = Path("."),
) -> list[str]:
    """Generate a minimal runnable SPA template in deterministic mode.
    
    Returns:
        List of relative file paths created (e.g., ['generated_apps/todo/server.js', ...])
    """
```

## 调用时机

在 `growth_dev/team/agents.py::run_coder()` 中：
```python
if context.executor == "codex":
    return _codex_stage_output("coder", context)
else:
    # deterministic fallback: generate minimal app template
    from .app_generation import generate_deterministic_app_files, is_app_generation_domain
    
    if is_app_generation_domain(context.domain):
        contract = _read_json_artifact(context, "app_contract.json")
        prd_text = (context.run_dir / "input_prd.md").read_text(encoding="utf-8", errors="replace")
        inputs = context.inputs if isinstance(context.inputs, dict) else {}
        app_slug = str(inputs.get("app_slug") or contract.get("app_slug") or "generated-app")
        
        files_changed = generate_deterministic_app_files(
            run_dir=context.run_dir,
            app_slug=app_slug,
            prd_text=prd_text,
            contract=contract,
            repo_root=context.repo_root,
        )
        
        record = {
            "status": "completed",
            "executor": "deterministic",
            "files_changed": files_changed,
            "tests_run": [],
            "risk_events": [],
            "blockers": [],
        }
        return AgentOutput(
            status="completed",
            output_paths=[context.write_json("code_run_record.json", record)],
            risk_events=[],
            summary="generated deterministic app template",
        )
    # ... existing deterministic logic for other domains
```

## 验证标准

- `node --check generated_apps/{app_slug}/server.js` 语法检查通过
- `node generated_apps/{app_slug}/server.js` 能启动服务器
- `curl http://127.0.0.1:8788/` 返回 HTML
- 浏览器打开后能看到空状态 UI
- 浏览器控制台 `window.app.state` 可访问
- `localStorage` 中能看到 `{app_slug}-state` 键

## 与 Codex 路径对比

| 维度 | Deterministic | Codex |
|---|---|---|
| 依赖 | 零外部依赖 | 需要 codex 二进制 |
| 质量 | 最小模板 | 根据 PRD 定制实现 |
| 速度 | 毫秒级 | 秒到分钟级 |
| 适用场景 | 环境验证、快速演示、CI | 真实生成任务 |
| 文件数 | 5 个固定文件 | 动态决定 |
