Code Agent 能力边界：职责可生长，运行期不可逾越

Code Agent 能外溢多远，取决于其产物能被自动验证 / 回滚 / 审计到多远——边界由产物的可验证性决定，不由模型能力决定。它越强越往上长成「编译 / 造件 / 蒸馏」的生产者；唯独运行期的事实计算与执行，永远落在共享 Runtime。

全链路极简架构图

Code Agent 的边界不取决于它有多强，而取决于产物是「会被固化复用的资产」还是「被当作事实的一次性结论」。前者越强越该多做，后者再强也不交给它。

Code Agent 能力边界与外溢前置条件

核心边界公式：Code Agent 能外溢多远 = 其产物能被自动验证 / 回滚 / 审计到多远。每一段外溢都有门禁，满足才允许接管，不满足则回退到默认态。

职责段 默认态 外溢前置条件（满足才允许 Code Agent 接管） 不满足时回退
① 编译 spec 人工 / 模板编译 输入有结构化 schema；编译产物有输出 schema 与门禁；产物作为「提议」经 review 才采纳 退回人工 / 模板编译
② 造模板 · 造端点 预制组件库 / 兜底定制 产物有契约测试；进入版本化资产库；端点强制 input/output schema + evidence；验证通过才晋升确定资产 退回一次性兜底定制
③ 蒸馏 Skill 人工沉淀 运行期 trace/evidence/run record 完整可回放；有 eval/regression 防退化；离线产出经回归测试才上线 保持散装服务端点，不升级
运行期执行 Runtime 确定性执行 禁区，不可外溢：运行期不得用 LLM 实时产出事实结论或替代确定 pipeline 永远落在共享 Runtime

通用总闸（四条同时成立才允许外溢）：① 可验证 schema+gate+test ② 可回滚 版本化·隔离 worktree ③ 可审计 diff/review/evidence ④ 默认提议、经门禁与 human-confirmed apply 才采纳。

模块职责

TDS Builder

把 PRD、Capability、AppSpec、EvalSpec 统一编译成单一交付包 TDS Bundle。

应用装配器

按确定性分流：模板装配做确定结构、runtime 绑定接业务逻辑、Code Agent 造件或兜底。

App 壳 (A)

自包含的 UI / 页面 / 状态机。可漂移、可重生成、可离线预览，不是事实源。

Runtime 服务 (B)

运行实体共享的业务能力，承载执行、证据、回放；先以服务端点上线，再沉淀为 Skill Bundle。

Code Agent 职责带

越强越向上生长——编译 spec、造模板与服务端点、蒸馏 Skill，覆盖生成 / 编译 / 沉淀。

运行期禁区

运行期的事实计算与执行永远落在 Runtime，不交概率组件；蒸馏属离线编译，为例外。

设计原则

A/B 切分

壳可漂移，能力单一真相。

多载体装配

确定走模板，逻辑走 runtime，定制才 codegen。

先服务后 Skill

演进而非一次性抽象成 bundle。

契约前置

端点 input/output 有 schema，结论挂 evidence。

闭环沉淀

反馈回写 Runtime，不从代码逆向 distill。

可生长 · 有禁区

Code Agent 生成期越强越好，运行期不碰事实执行。

一句总结：Code Agent 越强，越应成为确定性资产的生产者（编译 / 造件 / 蒸馏），覆盖生成→编译→沉淀全段；唯独运行期的事实执行，是它不可逾越的禁区，永远落在共享 Runtime。
