# User Story Draft Template

Use this template to turn product understanding into candidate user stories. User stories are planning aids; acceptance criteria remain the official testable contract.

## Story Format

`US-001`

- Card: As a `<role>`, I want `<capability>`, so that `<value>`.
- Conversation:
  - Context:
  - Workflow:
  - Edge cases:
  - Dependencies:
- Confirmation:
  - Acceptance criteria ids:
  - Verification command:
  - Evidence artifact:

## 3 C Checklist

- Card is concise and names role/action/value.
- Conversation captures assumptions and open questions.
- Confirmation maps to observable acceptance criteria.

## INVEST Checklist

- Independent enough to plan without unrelated refactors.
- Negotiable while still preserving the user outcome.
- Valuable to a named user/operator.
- Estimable from current capability boundary.
- Small enough for one or more bounded slices.
- Testable through public behavior or explicit artifact evidence.

## Quality Notes

- Do not turn assumptions into facts.
- Do not add implementation details unless they define user-observable behavior.
- Do not create stories without linked acceptance criteria.
